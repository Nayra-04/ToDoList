# To Do List – Manual Testing Project

## Project Overview
Manual testing project for a simple To Do List web application.
The goal is to practice writing test scenarios, executing tests, and reporting bugs professionally.

## What I Did
- Analyzed application features
- Wrote test scenarios
- Executed test cases manually
- Compared Expected vs Actual results
- Reported bugs in structured format
- Organized work using Trello

## Tools Used
- Manual Testing
- Trello (task & bug tracking)
- GitHub (documentation)

  
## Test Execution
- Functional testing
- UI testing
- Basic edge case testing (e.g., long task text)

## Bugs Found
- Incorrect items left counter
- Cannot mark long task as completed

## Purpose
This project demonstrates manual testing skills, bug reporting, and organized workflow management using Trello.
